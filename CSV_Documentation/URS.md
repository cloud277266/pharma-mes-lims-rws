# User Requirement Specification (URS)

## 개요
본 문서는 프로젝트(Mini LIMS, RWS Simulator, MES Dashboard)의 핵심 사용자 요구사항을 정의합니다. 특히 데이터 무결성(Data Integrity)과 관련된 규제 요구사항(GMP/CSV)을 충족하기 위한 기능적 명세를 포함합니다.

## 핵심 요구사항

### 1. Mini LIMS (실험정보관리시스템)
*   **[REQ-LIMS-001] Audit Trail (감사 증적)**: 시스템은 모든 품질 데이터(실험 결과 등)의 생성, 수정, 삭제 시 변경 사항을 영구적으로 기록해야 한다. 데이터 변경 시 반드시 '변경 사유(changeReason)'를 입력받아야 하며, 이를 누락할 경우 시스템은 변경을 거부해야 한다.
*   **[REQ-LIMS-002] Maker-Checker 워크플로우**: 실험 데이터의 최종 승인은 데이터 입력자(Maker)가 아닌 독립적인 권한을 가진 승인자(Checker)에 의해 이루어져야 한다.

### 2. RWS Simulator (칭량관리시스템)
*   **[REQ-RWS-001] 허용 오차 제어 (Hard-lock)**: 원료 칭량 시, 설정된 목표 중량의 허용 오차 한계(예: ±1%)를 벗어날 경우 시스템은 작업 완료 및 라벨 발행 진행을 시스템적으로 차단하고 경고를 발생시켜야 한다.
*   **[REQ-RWS-002] 바코드 검증**: 작업자는 칭량 전 원료의 바코드를 스캔하여 작업 지시서의 원료와 일치하는지 확인해야 하며, 불일치 시 칭량 작업을 시작할 수 없어야 한다.
